The Architecture

The challenge at this scale wasn't just volume‚Äîit was the variety of data types and the velocity required for decision making. We manage a data estate that grows daily, currently housing over 53 Billion rows of data.

To handle this, we implemented a robust Distributed Computing model using the Hadoop ecosystem on Google Cloud:

  • Hadoop & YARN (Cluster Management)

    We utilize Google Cloud Dataproc to provision ephemeral clusters. At the core, YARN (Yet Another Resource Negotiator) acts as the operating system for our big data applications, allocating resources (CPU/RAM) across worker nodes. This allows us to scale processing power dynamically based on the workload, ensuring cost-efficiency without sacrificing performance.

  • PySpark & Distributed Processing

    Standard SQL couldn't handle our complex unstructured logs. We switched to PySpark to leverage:

    • In-Memory Computation: Unlike MapReduce which writes to disk after every step, Spark keeps intermediate data in RAM, speeding up iterative algorithms by 100x.
    • Catalyst Optimizer: Spark constructs a Directed Acyclic Graph (DAG) of our queries. The Catalyst optimizer then rearranges operations (e.g., pushing filters down before joins) to generate the most efficient execution plan automatically.

  • Modern Transformation with Dataform

    For our BigQuery data warehouse, we treat "Data as Code." We integrate Google Cloud Dataform to manage SQL transformation pipelines. This allows us to apply software engineering best practices‚Äîsuch as version control (Git), testing, and CI/CD‚Äîdirectly to our SQL logic. Dataform handles dependency management, ensuring that tables are updated in the correct order and protecting downstream dashboards from broken data.

  • Decoupled Storage (GCS vs HDFS)

    We decoupled storage from compute. Instead of HDFS, we use Cloud Storage. This allows us to shut down Dataproc clusters (saving money) while persisting data in low-cost object storage, formatted as highly compressed Parquet files.

GDPR Compliance & Privacy Engineering

Operating in the European market means privacy is not an afterthought‚Äîit is the foundation of our architecture. We are strictly GDPR compliant and employ a "Privacy by Design" approach.

Consent-Based Tracking: We do not expose or share any user information with third-party software unless the user has explicitly accepted cookies. Our pipelines respect these consent signals at the ingestion layer.

Pseudonymization at Scale: For internal analytics, all Personally Identifiable Information (PII) is masked using cryptographic salting before it ever reaches the data lake. This ensures that while our data scientists can analyze aggregate behavior and campaign effectiveness, they never access raw user identities.
